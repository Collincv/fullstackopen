** 1.D - A more complex state, debugging React apps
 - Complex state - multiple "pieces" of state.
 - two consts for each beginning with useState(0)
 - alternate, one piece of state with object with a property for each button.  (both property values  will need
   to be updated on either click.).  Can update the state of the pieces that dont change with object spread.
 - it is forbidden in React to mutate state directly
 - Changing state has to always be done by setting the state to a new object
 - Storing all of the state in a single state object is a bad choice for the left and right button example.
 - Arrays (all clicks concatenated with 'R' or 'L') and displayed on the bottom
 - state update in React happens asyncchronously,(button count example).  using updated value to set total and set new value for new state
 - conditional rendering - movinv allClicks to history component
 - Debugging - add debugger anywhere in code for browser to display in debugger mode.  Components tab. React extension
 - Rules of Hooks - (*what is a hook specifically?* useState function must not be called from inside a loop, a conditional expression, or any place that is not a function defining a component.  rule ensures that hooks are always called in the same order).  It is defined inside a function body that defines a React component'
